## Session themes

1. Normalizer rule added for Word.Field casing (after VBA IDE
   demoted "Dim fld As Word.Field" to "Dim fld As Word.field"
   in basStyleInspector). Run-once across src/ fixed 4 cases
   across 3 files.
2. AuditHyperlinkStyling extended to check Font.Name=Carlito and
   Font.Size=9 in addition to color + underline. Caught a
   size=11 anomaly on a hyperlink inside an AuthorListItemTab
   paragraph - paragraph inherits its size, built-in Hyperlink
   style doesn't pin font/size.
3. Architectural decision: replace use of Word's built-in
   Hyperlink character style with a custom "BookHyperlink"
   style that pins all four font properties explicitly. Built-in
   Hyperlink cannot enforce font/size (inherits from paragraph);
   custom style is fully self-contained.
4. BookHyperlink implementation:
   - DefineBookHyperlinkStyle in basFixDocxRoutines.
   - Added to approved array; AuditOneStyle row added.
   - LockBookHyperlinks replaces LockHyperlinksToPalette
     (three-step: migrate built-in -> BookHyperlink, unlink
     active Hyperlinks, force-apply properties on all
     BookHyperlink runs). Built-in Hyperlink and FollowedHyperlink
     style definitions deliberately NOT touched.
   - LockHyperlinksAlwaysBlue alias removed (was due).
   - AuditHyperlinkStyling renamed AuditBookHyperlinkStyling.
   - EDSG/01-styles.md "Companion rule" rewritten; per-installation
     "Disable URL auto-format" recommendation added.
5. Test 17 (CountActiveHyperlinks) unchanged - still measures
   total Hyperlinks collection count across stories. Expected 0.

9. NEW RULE established 2026-05-15: all functions wired into a
   RUN_THE_TESTS slot must live inside aeBibleClass.cls (typically
   as Private Function). Module-side implementations are a
   divergence; if a function is also useful from outside the class,
   keep the canonical body in the class and add a thin stub in the
   module that delegates in. Config/data SSOTs (e.g.
   GetApprovedStyles) are not test code and may remain in modules.
   Saved as memory feedback_test_functions_in_class.md.

   currently dispatches AuditNonPaletteStyleColors directly from
   basStyleInspector. That body needs to move into aeBibleClass.cls
   with a delegate stub left in basStyleInspector for the standalone
   Immediate-window call.

## Session themes (continued) -- Item 13 / 2.4 + architecture rule

11. Class-encapsulation rule formalised. The earlier
    test-functions-in-class rule was broadened: any class-related
    code stays inside the class, classes may call other classes
    (aeAssertClass, aeLoggerClass, aeBibleCitationClass, etc.),
    and stateless specs / lookup tables / config SSOTs remain in
    modules. Module/class file extension doubles as a permission
    signal for casual contributors - safe-to-edit vs ask-first.
    basBiblePalette explicitly stays a module (stateless lookup
    table; converting would dissolve the safety signal).

    Memory rule renamed feedback_class_encapsulation.md.
    Rationale appended to rvw/Code_review 2026-05-15.md as new
    § 9. Contributor-facing version written as
    EDSG/12-module-vs-class.md and linked from EDSG/README.md.
