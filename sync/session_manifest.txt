A. CHARACTER-STYLE AUDIT TAXONOMY GAP CLOSED (2026-05-31)
   - `CountAuditStyles_ToFile` (Test 49) walks paragraph styles
     only and is structurally blind to character styles riding
     on runs INSIDE paragraphs (e.g. "Chapter Verse marker" and
     "Verse marker" inside VerseText). Operator surfaced the gap:
     "VerseText = Chapter Verse marker = Verse marker = 31,102,
     only VerseText is validated."
   - SSOT decision: do NOT split GetApprovedStyles into separate
     para/char lists. Word itself knows each style's `.Type`;
     materialize the taxonomy at runtime via a new helper that
     filters by `WdStyleType`. Single source of truth preserved.
   - basTEST_aeBibleConfig.bas: new
     `Public Function GetApprovedStylesByType(wantType As WdStyleType)
     As Variant`. Walks `GetApprovedStyles()`, asks Word for each
     style's `.Type`, returns matching names. Missing styles skip
     silently (PromoteApprovedStyles already reports them).
