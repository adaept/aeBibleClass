# 06 — Internationalization (i18n)

Status: skeleton. The Study Bible is initially produced in English;
this page captures what's known about preparing the system for
locales beyond English.

## Open question: first locale

Not yet decided. The shape of this page changes substantially based
on the first localization target:

- **Western European** (French, Spanish, German) — accented Latin,
  same script direction, similar typographic conventions. Lowest
  friction.
- **Cyrillic** (Russian, Ukrainian) — different script, fonts must
  cover the range.
- **CJK** (Chinese, Japanese, Korean) — different script, different
  line metrics, vertical text considerations, different paragraph
  break semantics.
- **RTL** (Hebrew, Arabic) — right-to-left bidi, mirrored layout for
  many UI conventions, different list/numbering conventions.

Once the first target is chosen, this page expands into specifics.

## Style implications

Each approved style in [01-styles](01-styles.md) carries a font name.
For each locale:

- **Replace** the font with one that covers the target script.
- **Verify fallback chains** — what happens when a glyph isn't in
  the primary font? Word's font fallback may pick a non-matching
  font and look bad.
- **Audit ParagraphFormat metrics** — line spacing tuned for English
  may be wrong for languages with different x-height or stroke
  density.

Approved styles that may need locale-specific overrides:

- `VerseText` — **primary translation target.** Every Bible
  verse uses this style since the 2026-05-01 migration; locale
  font and metrics changes land here first.
- `BodyText`, `Heading 1`, `Heading 2` — residual reading flow
  (front matter, chapter intros, headings). `BodyTextIndent` was
  removed from the approved array during the VerseText
  migration.
- `AuthorBodyText`, `AuthorSectionHead`, `AuthorBookRef` — commentary
  uses a different font family from the Bible body to signal
  commentary; that contrast must remain in the localized fonts too.
- `Words of Jesus`, `EmphasisRed`, `EmphasisBlack` — character styles.
  Color choices may carry locale-specific meaning.
- Headers / footers — see [05-headers-footers](05-headers-footers.md).

## Process expectations (sketch)

For each new locale:

1. Fork or branch the source `.docm` and the `RUN_TAXONOMY_STYLES`
   constants.
2. Substitute fonts; verify with `DumpAllApprovedStyles`.
3. Re-run `ListApprovedStylesByBookOrder` against the localized
   document — the book order should be the same, but page numbers
   shift as text length changes.
4. Re-validate the `approved` array against the locale-specific
   book order.
5. Run `SUPER_TEST_RUNS` (when available) for the full QA gate.
